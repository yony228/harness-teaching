# 技术摘要分析技能

## 使用场景

- 对采集的 GitHub Trending 原始数据进行深度分析与价值评估
- 从原始列表中提取有价值的技术项目，精炼为可读性强的技术摘要
- 生成结构化分析结果供知识库归档和多渠道分发使用
- 触发条件：每次 raw 数据采集完成后自动触发

## 执行步骤

### 步骤 1：读取待分析原始数据

使用 `Read` 和 `Glob` 定位并读取最近的采集文件：

```
Glob: knowledge/raw/github-trending-*.json
```

选择日期最新的文件，读取其 `items` 数组作为分析输入。

### 步骤 2：逐条深度分析

对每条原始数据执行结构化分析，生成以下字段：

- **摘要**：不超过 50 字，精炼概括项目核心价值
- **技术亮点**：列出 2-3 个关键技术亮点，**必须用事实数据支撑**（如 benchmarks、性能指标、独特架构、引用量、采用率等），禁止空泛描述
- **评分**：1-10 分，附评分理由
- **标签建议**：补充原始数据可能缺失的精准标签

```
评分标准：
- 9-10 分：改变格局（技术范式级突破，可能重塑行业格局）
- 7-8 分：直接有帮助（对开发者/团队有明确实用价值）
- 5-6 分：值得了解（有阅读价值但落地尚需时）
- 1-4 分：可略过（概念性内容或价值有限）
```

### 步骤 3：趋势发现

在全部条目中识别整体层面的趋势信号：

- **共同主题**：哪些技术方向出现频率最高？
- **新概念/新名词**：首次出现或新兴的技术概念是什么？
- **生态信号**：是否有大厂/明星项目跟进？是否与已有主流技术形成互补或替代关系？

输出趋势洞察数组，供知识文档撰写使用。

### 步骤 4：输出分析结果 JSON

生成文件路径：

```
knowledge/articles/tech-summary-YYYY-MM-DD.json
```

格式要求：

```json
{
  "source": "github_trending",
  "skill": "tech-summary",
  "analyzed_at": "YYYY-MM-DDTHH:MM:SSZ",
  "summary_count": 数字,
  "trends": [
    {
      "theme": "主题名称",
      "description": "趋势描述",
      "weight": "high/medium/low"
    }
  ],
  "items": [
    {
      "name": "owner/repo",
      "url": "https://github.com/owner/repo",
      "summary": "50字以内摘要",
      "highlights": [
        {
          "point": "亮点描述",
          "fact": "支撑事实（数据/案例/指标）"
        }
      ],
      "score": 数字,
      "score_reason": "评分理由",
      "tags": ["tag1", "tag2"]
    }
  ]
}
```

## 注意事项

- **分数约束**：15 条原始数据中，9-10 分的条目**不得超过 2 个**；如无符合项，则最高分不超过 8 分
- **事实驱动**：技术亮点描述**必须**有据可查（性能数据、 adopion 指标、社区反应、引用来源），禁止主观空泛好评
- **摘要精炼**：中文摘要严格控制在 50 字以内，去掉冗余修饰词，直击核心价值
- **评分一致性**：评分理由需与评分对应（如给 9 分但理由仅为"Star 多"则不符合标准）
- 分析结果需通过 Pydantic schema 校验，不合格数据标记为 `invalid` 并写入 `knowledge/articles/` 下
- 分析过程中的日志记录必须包含被过滤或降级的条目原因
